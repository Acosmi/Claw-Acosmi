# 小红书运营 SOP

用于小红书图文发布和互动管理。当前系统走 Cookie + 浏览器自动化模式，不支持直接输入账号密码登录，但已支持在媒体管理页一键打开登录页并自动采集 Cookie。

## 如何登录 / 授权

### 当前接入方式

1. 在媒体管理页进入“小红书”账号卡片
2. 点击“打开登录页”，系统会自动拉起小红书创作者页
3. 在浏览器完成扫码 / 登录
4. 点击“检查并保存登录态”，系统会自动落 Cookie 到当前账号配置
5. 建议补充账号名称 / 账号 ID，便于区分多账号

### 当前未内置的能力

- 没有账号密码登录表单
- 没有系统内嵌扫码弹窗（仍在外部浏览器完成扫码）

所以现在的“登录”本质上是“外部浏览器完成扫码，系统自动采集并绑定 Cookie”。

## 如何看规则

发布前至少检查：

- 标题建议 `<= 20` 字
- 正文建议 `<= 1000` 字
- 图片 `<= 9` 张
- 标签建议 `3-5` 个
- 操作间隔 `>= 5` 秒
- Cookie 必须有效

## 内容风格

- 口语化
- 信息密度高
- 每段不要太长
- 结尾有互动引导
- 不要太像公文或官网稿

## 标准发布流程

```text
1. content_compose 创建小红书草稿
2. 人工审批
3. media_publish 查询草稿状态，确认 approved
4. media_publish 发布到小红书
5. 若失败，优先检查 Cookie、页面结构、截图目录
```

## 互动流程

### 评论

- 查看评论
- 回复评论
- 遇到争议内容上报主智能体

### 私信

- 查看私信
- 回复私信
- 避免发送外链和明显导流内容

## 风控规则

- 单次操作之间保留足够间隔
- 不连续高频发布
- 不在评论区堆联系方式
- 不回复明显恶意内容
- 出现异常优先停下来查 Cookie 和页面变化

## 常见问题

| 问题 | 可能原因 | 处理 |
|---|---|---|
| 无法发布 | Cookie 过期 | 重新执行“打开登录页” + “检查并保存登录态” |
| 互动失败 | 页面结构变化 / 风控 | 看错误截图与日志 |
| 频繁报错 | 操作太快 | 提高 rate limit |

## 建议输出

每次执行后至少说明：

- 目标账号
- 标题
- 使用的标签
- 发布是否成功
- 若失败，失败阶段和原因
- 若有互动，回复了哪些评论 / 私信
