# 微信公众号运营 SOP

用于微信公众号内容生产与发布。当前系统走官方 API 路径，不是浏览器模拟登录。

## 如何接入

### 当前授权方式

- 需要填写 `AppID`
- 需要填写 `AppSecret`
- 可选填写 `tokenCachePath`
- 不需要在系统里做网页扫码登录

### 配置完成标准

- 发布器已启用
- `AppID` 不为空
- `AppSecret` 不为空
- 媒体管理页显示“已配置”

## 如何看规则

发布前至少检查以下限制：

- 标题建议 `<= 64` 字
- 正文建议结构化长文
- 正文最终要转成 HTML
- 封面图必须可用
- 图片格式建议 JPG / PNG
- 图片大小不要超过 1MB
- 发布是异步的，需要查状态

## 标准发布流程

```text
1. content_compose 创建公众号草稿
2. 返回草稿给主智能体 / 人工审批
3. media_publish 查询草稿状态，确认 approved
4. media_publish 执行 publish
5. 轮询发布状态直到 published / failed
```

## 草稿要求

### 推荐结构

```text
标题
导语
一、问题/背景
二、分析/方法
三、结论/建议
总结 + CTA
```

### 内容规范

- 适合 1500-3000 字
- 分段清晰，避免大段文字墙
- 重要结论前置
- 引用数据时说明来源
- 避免标题党

## 发布前检查清单

- 标题长度合规
- 正文没有占位词
- 关键事实已核实
- 有封面图
- 图片与正文匹配
- 草稿状态已批准

## 常见错误

| 问题 | 可能原因 | 处理 |
|---|---|---|
| access_token 无效 | AppSecret 错误或 token 过期 | 重新检查配置 |
| 发布失败 | 草稿未审批 / 图片不合规 | 先查 status 和素材 |
| 无法创建图文 | HTML 或素材异常 | 检查正文和封面 |

## 建议输出

完成公众号内容时，至少回报：

- 目标主题
- 标题
- 内容摘要
- 关键配图说明
- 是否已进入待审批
- 若能发布，返回 publish_id / 状态
