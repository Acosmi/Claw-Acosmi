# 热点追踪与选题分析

适用于媒体运营子智能体的热点发现阶段。目标不是“看见热搜”，而是把热搜转成可发布、可差异化、可执行的选题。

## 数据源

| 源 | 适合什么内容 | 注意点 |
|---|---|---|
| 微博热搜 | 时效强、事件快 | 娱乐社会类噪音高 |
| 百度热搜 | 综合搜索需求 | 更适合大众话题 |
| 知乎热榜 | 深度讨论、知识型话题 | 适合公众号长文 |

## 标准工作流

### 1. 先看源是否正常

- 优先确认微博 / 百度 / 知乎是否都在工作。
- 某个源异常时，不要直接放弃整轮分析，先用其他源补齐。

### 2. 拉取热点

建议先全量，再按源精筛：

```text
trending_topics { action: "list_sources" }
trending_topics { action: "fetch", limit: 30 }
trending_topics { action: "fetch", source: "baidu", category: "tech", limit: 20 }
```

### 3. 过滤垃圾热点

直接丢弃以下主题：

- 与账号定位无关
- 争议极强但无法核实
- 只有情绪价值，没有内容承接点
- 热度极高但平台已有大量同质内容

### 4. 给出可执行选题

每个保留话题至少产出：

- 话题标题
- 热度判断
- 推荐平台
- 创作角度 2-3 个
- 风格建议（casual / informative / professional）

## 平台匹配规则

| 场景 | 推荐平台 | 理由 |
|---|---|---|
| 深度分析 / 政策解读 / 行业洞察 | 微信公众号 | 能承载长文和结构化论证 |
| 种草 / 教程 / 经验分享 / 视觉型内容 | 小红书 | 图文传播和互动更强 |
| 官方公告 / 品牌长文 / SEO 内容 | 自有网站 | 可沉淀为站内资产 |

## 选题判断标准

### 值得做

- 热点和账号定位强相关
- 有明确受众
- 能在 2 小时内组织出完整内容
- 可以做出与竞品不同的切口

### 不值得做

- 只有标题，没有论点
- 必须依赖未核实消息
- 一看就会踩平台红线
- 明显会把账号带偏

## 输出格式建议

每轮至少交付 3 条：

1. 话题名 + 热度
2. 目标平台
3. 推荐标题方向
4. 内容骨架
5. 风险提醒

## 常见动作

### 查某个源

```text
trending_topics { action: "fetch", source: "weibo", limit: 20 }
```

### 查科技类百度热点

```text
trending_topics { action: "fetch", source: "baidu", category: "tech", limit: 20 }
```

### 深挖某个主题

```text
trending_topics { action: "analyze", topic: "AI Agent 商业化" }
```

## 风险控制

- 不要把不同平台的热度值直接横向比较
- 对事实性内容，进入创作前补一次外部核实
- 热点窗口极短，输出要偏“能马上写”，而不是“分析很全但落不了地”
